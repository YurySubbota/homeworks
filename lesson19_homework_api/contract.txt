Цель: Написать веб-приложение на Flask, которое позволяет добавлять,
удалять и получать информацию о продуктах питания из базы данных PostgreSQL.

Требования:

Использовать библиотеки sqlalchemy, psycopg2 и flask для работы с базой данных и веб-сервером.
Создать таблицу products в базе данных products с полями id (первичный ключ, целое число),
name (название продукта, строка, не может быть пустым), calories (калорийность, целое число),
protein (белки, целое число), fat (жиры, целое число) и carbs (углеводы, целое число).
Написать функции add_product, delete_product и get_product, которые принимают в качестве
аргумента словарь с данными о продукте или его название и выполняют
соответствующие операции с базой данных (добавление, удаление или выборка записи).
Написать декоратор app.route, который обрабатывает запросы к адресу /products с методами GET и POST.
Если запрос с методом GET содержит параметр name, то вызвать функцию
get_product с этим параметром и вернуть результат в формате JSON с кодом 200, если продукт найден,
или с сообщением “Product not found” с кодом 404, если продукт не найден.
Если запрос с методом GET не содержит параметр name, то вернуть сообщение “Name parameter is required” с кодом 400.
Если запрос с методом POST содержит данные в формате JSON с полем name,
то вызвать функцию get_product с этим полем и проверить, существует ли уже продукт с таким названием в базе данных.
Если продукт существует, то вернуть сообщение “Product already exists” с кодом 409. Если продукт не существует,
то вызвать функцию add_product с данными из запроса и вернуть сообщение “Product added successfully” с кодом 201.
Если запрос с методом POST не содержит данные в формате JSON или не содержит поле name,
то вернуть сообщение “Invalid data” с кодом 400.
